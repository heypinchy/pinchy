DigitalOcean is a solid choice for running Pinchy — straightforward interface, predictable pricing, and data centers on multiple continents. If your team is distributed or you want to keep your AI agents close to your users, DigitalOcean's global footprint makes that easy.

## Why DigitalOcean for self-hosted AI agents?

- **Global data centers** — New York, San Francisco, London, Amsterdam, Frankfurt, Singapore, Bangalore, Sydney, and more. Pick the region closest to your team.
- **Simple pricing** — A Droplet with 4 GB RAM starts at around $24/month with predictable costs.
- **Team accounts** — Built-in team management for sharing infrastructure access.

<Aside type="note">
  For EU data residency requirements, choose the **Amsterdam (AMS3)** or **Frankfurt (FRA1)** region. Your AI conversations and data will stay in the EU.
</Aside>

## What you'll need

- A [DigitalOcean account](https://cloud.digitalocean.com/registrations/new) (free to create, pay-as-you-go)
- A domain name (optional for testing, required for production HTTPS)
- An LLM provider API key (Anthropic, OpenAI, or Google) — you'll enter this in Pinchy's setup wizard

## Create your Droplet

A "Droplet" is DigitalOcean's name for a virtual server.

<Steps>

1. **Log in to the [DigitalOcean Control Panel](https://cloud.digitalocean.com/)**

   Click **Create → Droplets**.

2. **Configure the Droplet**

   | Setting | Value |
   | ------- | ----- |
   | **Region** | Amsterdam or Frankfurt (EU) / New York or San Francisco (US) |
   | **Image** | Ubuntu (latest LTS version) |
   | **Size** | The smallest plan with at least 4 GB RAM and 2 vCPUs |
   | **Hostname** | `pinchy` (or whatever you prefer) |

   <Aside type="tip">
     For teams with more than 5 users or heavy agent workloads, consider the $48/mo plan (8 GB RAM, 4 vCPUs) for smoother performance.
   </Aside>

3. **Choose authentication**

   Under **Authentication**, you can choose **SSH Key** or **Password**. For the automated setup below, either works — you only need SSH access later for production hardening.

4. **Paste the setup script**

   Scroll down to **Advanced Options** and check **Add User Data**. Paste the following script. It automatically installs Docker, deploys Pinchy, sets up a firewall, and adds swap.

   ```yaml
   #cloud-config
   package_update: true

   packages:
     - docker.io
     - docker-compose-v2
     - git
     - ufw

   runcmd:
     - systemctl enable docker
     - systemctl start docker
     - ufw allow OpenSSH
     - ufw allow 80/tcp
     - ufw allow 443/tcp
     - ufw --force enable
     - fallocate -l 2G /swapfile
     - chmod 600 /swapfile
     - mkswap /swapfile
     - swapon /swapfile
     - echo '/swapfile none swap sw 0 0' >> /etc/fstab
     - mkdir -p /opt/pinchy-loading
     - curl -fsSL https://docs.heypinchy.com/installing.html -o /opt/pinchy-loading/index.html || true
     - sed -i "s/INSTALL_START_TIME/$(date +%s)000/" /opt/pinchy-loading/index.html
     - cd /opt/pinchy-loading && python3 -m http.server 80 &
     - echo $! > /tmp/loading-server.pid
     - git clone https://github.com/heypinchy/pinchy.git /opt/pinchy
     - cd /opt/pinchy && git checkout %%PINCHY_VERSION%%
     - echo 'PINCHY_PORT=80' > /opt/pinchy/.env
     - cd /opt/pinchy && docker compose build
     - kill $(cat /tmp/loading-server.pid) 2>/dev/null || true
     - rm -rf /opt/pinchy-loading /tmp/loading-server.pid
     - cd /opt/pinchy && docker compose up -d
   ```

   <details>
   <summary>What does this script do?</summary>

   This is a **cloud-init** script — a standard way to configure servers on first boot. When your Droplet starts, it automatically:

   1. Installs Docker, Git, and a firewall
   2. Opens only the ports needed (SSH, HTTP, HTTPS)
   3. Adds 2 GB of swap space (extra memory from disk — important for 4 GB Droplets)
   4. Shows a loading page while Pinchy is being built
   5. Downloads, builds, and starts Pinchy

   No manual setup required. You can also [download the script](/cloud-init.yml) separately.
   </details>

5. **Create the Droplet**

   Click **Create Droplet**. It'll be ready in about 30 seconds, but the setup script needs another **5–8 minutes** to finish installing everything (DigitalOcean builds tend to be a bit slower than other providers).

6. **Note the IP address**

   The Droplet's IPv4 address appears on the dashboard (e.g., `164.90.x.x`).

7. **Open Pinchy**

   After about 5 minutes, visit `http://<your-droplet-ip>` in your browser. You should see the Pinchy setup wizard.

   <Aside type="note">
     If you see a connection error, wait another minute — the build might still be running. You can check the progress by connecting via SSH (`ssh root@<your-droplet-ip>`) and running `docker compose -f /opt/pinchy/docker-compose.yml logs -f pinchy`.
   </Aside>

   The setup wizard will guide you through creating your admin account and configuring your first LLM provider.

</Steps>

## Production setup

The steps above get Pinchy running over plain HTTP. For production use, you should add HTTPS and pin your secrets. Follow the [VPS Deployment Guide — Production setup](/guides/vps-deployment/#production-setup) for:

- **HTTPS with Caddy** — automatic SSL certificates for your domain
- **Setting BETTER_AUTH_URL** — so login sessions work with your domain
- **Pinning secrets** — so your encryption keys don't depend on Docker volumes
- **Switching to port 7777** — once Caddy handles port 80/443, change Pinchy to run on 7777 internally

<details>
<summary>How do I connect to my Droplet to run these commands?</summary>

You'll need to use **SSH** (Secure Shell) — a way to run commands on a remote server from your terminal.

If you chose password authentication when creating the Droplet:

```bash
ssh root@<your-droplet-ip>
```

Enter the password you set (or check your email for the one DigitalOcean generated).

If you chose SSH key authentication, make sure you have the key on your machine, then:

```bash
ssh root@<your-droplet-ip>
```

<details>
<summary>Where do I find the terminal?</summary>

- **Mac** — Open **Terminal** (search for it in Spotlight with ⌘+Space)
- **Windows** — Open **PowerShell** (search in Start menu) or install [Windows Terminal](https://aka.ms/terminal)
- **Linux** — Open your terminal app (usually Ctrl+Alt+T)
</details>

While connected, every command you type runs on your Droplet. Type `exit` to disconnect.
</details>

## Point your domain (optional but recommended)

To use HTTPS, you need a domain pointing to your Droplet. Add a DNS record at your domain registrar (the service where you registered your domain):

| Type | Name | Value |
| ---- | ---- | ----- |
| A | `pinchy` (or your preferred subdomain) | Your Droplet's IP address |

<details>
<summary>What's an A record?</summary>

An A record is a DNS entry that maps a domain name (like `pinchy.example.com`) to an IP address (like `164.90.1.23`). When someone visits your domain, their browser looks up the A record to find your server.

Look for "DNS records" or "DNS settings" in your domain registrar's dashboard.
</details>

You can also manage DNS directly in DigitalOcean under **Networking → Domains**.

## DigitalOcean-specific tips

### Enable backups

DigitalOcean offers automated weekly backups for 20% of the Droplet cost. Enable this under **Droplet → Backups** — it's the easiest disaster recovery option.

### Cloud firewall

DigitalOcean offers **Cloud Firewalls** under **Networking → Firewalls**. For defense in depth, configure the cloud firewall to only allow ports 22, 80, and 443 — in addition to the host firewall (`ufw`) that the setup script already configured.

### Volumes for persistent data

For larger Knowledge Base deployments, attach a DigitalOcean Volume (up to 16 TB) and mount it as a data directory. See [Mount Data Directories](/guides/mount-data-directories/) for details.

### Scaling up

If you outgrow the 4 GB Droplet, you can resize it in the control panel. Pinchy's data lives in Docker volumes, so it survives Droplet resizes. Power off the Droplet first, resize, then power back on — downtime is typically under 2 minutes.
