Hetzner Cloud is an excellent choice for running Pinchy. Their servers run in EU data centers (Germany and Finland), which keeps your data under EU jurisdiction — no overseas transfers, no third-party cloud AI providers touching your conversations. Combined with strong price-to-performance ratios, it's the go-to option for European teams that take data sovereignty seriously.

## Why Hetzner for self-hosted AI agents?

- **EU data centers** — Falkenstein, Nuremberg, Helsinki. Your AI conversations stay in the EU.
- **Price** — A CX23 (4 GB RAM, 2 vCPUs) costs around €4/month. Enough to run Pinchy with a small team.
- **Predictable pricing** — No surprise bandwidth charges. 20 TB outbound traffic included.
- **ISO 27001 certified** — Relevant if your compliance team asks where the servers are.

## What you'll need

- A [Hetzner Cloud account](https://console.hetzner.cloud/) (free to create, pay-as-you-go)
- A domain name (optional for testing, required for production HTTPS)
- An LLM provider API key (Anthropic, OpenAI, or Google) — you'll enter this in Pinchy's setup wizard

## Create your server

<Steps>

1. **Log in to the [Hetzner Cloud Console](https://console.hetzner.cloud/)**

   Create a new project (e.g., "Pinchy") or use an existing one.

2. **Create a server**

   Click **Add Server** and configure:

   | Setting | Value |
   | ------- | ----- |
   | **Location** | Falkenstein (fsn1) or Nuremberg (nbg1) |
   | **Image** | Ubuntu 24.04 |
   | **Type** | CX23 (2 vCPUs, 4 GB RAM) — minimum for Pinchy |
   | **Name** | `pinchy` (or whatever you prefer) |

   <Aside type="tip">
     For teams with more than 5 users or heavy agent workloads, consider CX33 (4 vCPUs, 8 GB RAM) for smoother performance.
   </Aside>

3. **Paste the setup script**

   Before clicking "Create", scroll down to **Cloud config** and paste the following script. It automatically installs Docker, deploys Pinchy, sets up a firewall, and adds swap — no manual SSH needed.

   ```yaml
   #cloud-config
   package_update: true

   packages:
     - docker.io
     - docker-compose-v2
     - git
     - ufw

   runcmd:
     - systemctl enable docker
     - systemctl start docker
     - ufw allow OpenSSH
     - ufw allow 80/tcp
     - ufw allow 443/tcp
     - ufw allow 7777/tcp
     - ufw --force enable
     - fallocate -l 2G /swapfile
     - chmod 600 /swapfile
     - mkswap /swapfile
     - swapon /swapfile
     - echo '/swapfile none swap sw 0 0' >> /etc/fstab
     - git clone https://github.com/heypinchy/pinchy.git /opt/pinchy
     - cd /opt/pinchy && git checkout %%PINCHY_VERSION%%
     - cd /opt/pinchy && docker compose up --build -d
   ```

   <details>
   <summary>What does this script do?</summary>

   This is a **cloud-init** script — a standard way to configure servers on first boot. When your server starts, it automatically:

   1. Installs Docker, Git, and a firewall
   2. Opens only the ports needed (SSH, HTTP, HTTPS, and Pinchy's port 7777)
   3. Adds 2 GB of swap space (extra memory from disk — important for 4 GB servers)
   4. Downloads Pinchy and starts it

   No manual setup required. You can also [download the script](/cloud-init.yml) separately.
   </details>

4. **Create the server**

   Click **Create & Buy Now**. Your server will be ready in about 20 seconds, but the setup script needs another **3–5 minutes** to finish installing everything.

5. **Note the IP address**

   The server's IPv4 address appears on the overview page (e.g., `116.202.x.x`).

6. **Open Pinchy**

   After about 5 minutes, visit `http://<your-server-ip>:7777` in your browser. You should see the Pinchy setup wizard.

   <Aside type="note">
     If you see a connection error, wait another minute — the build might still be running. You can check the progress by connecting to your server via SSH (`ssh root@<your-server-ip>`) and running `docker compose -f /opt/pinchy/docker-compose.yml logs -f pinchy`.
   </Aside>

   The setup wizard will guide you through creating your admin account and configuring your first LLM provider.

</Steps>

## Production setup

The steps above get Pinchy running over plain HTTP. For production use, you should add HTTPS and pin your secrets. Follow the [VPS Deployment Guide — Production setup](/guides/vps-deployment/#production-setup) for:

- **HTTPS with Caddy** — automatic SSL certificates for your domain
- **Setting BETTER_AUTH_URL** — so login sessions work with your domain
- **Pinning secrets** — so your encryption keys don't depend on Docker volumes
- **Closing port 7777** — once HTTPS is set up, remove the direct access rule

<details>
<summary>How do I connect to my server to run these commands?</summary>

You'll need to use **SSH** (Secure Shell) — a way to run commands on a remote server from your terminal.

**First, you need an SSH key.** Check if you already have one:

```bash
cat ~/.ssh/id_ed25519.pub
```

If you get "No such file", create one:

```bash
ssh-keygen -t ed25519 -C "your-email@example.com"
```

Press Enter to accept defaults. Then add the key to Hetzner: **Security → SSH Keys → Add SSH Key**, and paste the output of `cat ~/.ssh/id_ed25519.pub`.

**To connect** (you may need to rebuild the server with the SSH key, or add it via the Hetzner rescue console):

```bash
ssh root@<your-server-ip>
```

<details>
<summary>Where do I find the terminal?</summary>

- **Mac** — Open **Terminal** (search for it in Spotlight with ⌘+Space)
- **Windows** — Open **PowerShell** (search in Start menu) or install [Windows Terminal](https://aka.ms/terminal)
- **Linux** — Open your terminal app (usually Ctrl+Alt+T)
</details>

While connected, every command you type runs on your server. Type `exit` to disconnect.
</details>

## Point your domain (optional but recommended)

To use HTTPS, you need a domain pointing to your server. Add a DNS record at your domain registrar (the service where you registered your domain):

| Type | Name | Value |
| ---- | ---- | ----- |
| A | `pinchy` (or your preferred subdomain) | Your server's IP address |

<details>
<summary>What's an A record?</summary>

An A record is a DNS entry that maps a domain name (like `pinchy.example.com`) to an IP address (like `116.202.1.23`). When someone visits your domain, their browser looks up the A record to find your server.

Look for "DNS records" or "DNS settings" in your domain registrar's dashboard.
</details>

## Hetzner-specific tips

### Snapshots for backups

Hetzner offers server snapshots (€0.012/GB/month). Take a snapshot after your initial setup — it lets you restore the entire server in minutes if something goes wrong.

**Servers → your server → Snapshots → Create Snapshot**

### Cloud firewall

Hetzner offers a **Firewall** feature under **Security → Firewalls**. For defense in depth, configure the cloud firewall to only allow ports 22, 80, and 443 — in addition to the host firewall (`ufw`) that the setup script already configured.

### Volumes for persistent data

For larger Knowledge Base deployments, you can attach a Hetzner Volume (up to 10 TB) and mount it as a data directory. See [Mount Data Directories](/guides/mount-data-directories/) for details.

### Scaling up

If you outgrow the CX23, you can resize to a larger server type directly in the Hetzner console. Pinchy's data lives in Docker volumes, so it survives server resizes. Plan for a brief downtime window (a few minutes) during the resize.
