All API endpoints require authentication unless noted otherwise. Unauthenticated requests receive a `401 Unauthorized` response. Authenticate by including a valid session cookie (set during login via the web UI).

Some endpoints are **admin only** ‚Äî non-admin users receive a `403 Forbidden` response.

## WebSocket (Chat)

Agent chat is handled over WebSocket at `/api/ws`, not via REST. The browser sends JSON messages and receives streaming responses. For details on the WebSocket protocol, message types, and session management, see the [Architecture](/architecture/#request-flow) page.

## Health

### `GET /api/health`

Service health check. **Public ‚Äî no authentication required.** Suitable for load-balancer probes and uptime monitoring.

**Response:** `200 OK` when Pinchy is running. `503 Service Unavailable` when upstream dependencies (database, OpenClaw) are unreachable.

### `GET /api/health/openclaw`

OpenClaw gateway reachability check. **Public ‚Äî no authentication required.**

Useful for monitoring the WebSocket gateway separately from the web process.

## Setup (public)

Setup endpoints handle the initial installation and first-admin account. They are accessible **only before setup has completed** ‚Äî once an admin exists, they return `409 Conflict`.

### `GET /api/setup/status`

Check whether setup is required.

**Response:**

```json
{
  "setupComplete": false,
  "providerConfigured": false
}
```

### `POST /api/setup`

Create the first admin account.

**Request body:**

| Field      | Type   | Required | Description                  |
| ---------- | ------ | -------- | ---------------------------- |
| `name`     | string | Yes      | Admin display name           |
| `email`    | string | Yes      | Admin email address          |
| `password` | string | Yes      | Admin password (min 8 chars) |

**Response:** `201 Created`. A session cookie is set ‚Äî the admin is logged in immediately.

### `POST /api/setup/provider`

Configure the first LLM provider as part of the setup flow.

**Request body:**

| Field      | Type   | Required | Description                                                     |
| ---------- | ------ | -------- | --------------------------------------------------------------- |
| `provider` | string | Yes      | `anthropic`, `openai`, `google`, `ollama-cloud`, `ollama-local` |
| `apiKey`   | string | Cond.    | Required for cloud providers                                    |
| `baseUrl`  | string | Cond.    | Required for `ollama-local`                                     |

**Response:** `{ "success": true }`

## Agents

### `GET /api/agents`

List agents visible to the current user. Admins see all agents. Non-admin users see shared agents and their own personal agents.

**Response:**

```json
[
  {
    "id": "uuid",
    "name": "HR Policy Assistant",
    "model": "anthropic/claude-haiku-4-5-20251001",
    "templateId": "knowledge-base",
    "allowedTools": ["pinchy_ls", "pinchy_read"],
    "pluginConfig": {
      "allowed_paths": ["/data/hr-policies"]
    },
    "isPersonal": false,
    "ownerId": "user-uuid",
    "createdAt": "2025-01-15T10:00:00.000Z",
    "updatedAt": "2025-01-15T10:00:00.000Z"
  }
]
```

### `POST /api/agents`

Create a new agent. The agent inherits default tool permissions from its template. Use `PATCH /api/agents/:id` to configure permissions after creation.

**Request body:**

| Field        | Type   | Required | Description                                    |
| ------------ | ------ | -------- | ---------------------------------------------- |
| `name`       | string | Yes      | Display name for the agent                     |
| `templateId` | string | Yes      | Template to use (`knowledge-base` or `custom`) |

**Example ‚Äî Knowledge Base agent:**

```json
{
  "name": "HR Policy Assistant",
  "templateId": "knowledge-base"
}
```

**Example ‚Äî Custom agent:**

```json
{
  "name": "General Assistant",
  "templateId": "custom"
}
```

**Response:** `201 Created` with the created agent object.

**Errors:**

- `400` ‚Äî missing or invalid `name` or `templateId`
- `401` ‚Äî not authenticated

### `GET /api/agents/:id`

Get a single agent by ID.

**Response:** The agent object, or `404` if not found.

### `PATCH /api/agents/:id`

Update an agent's settings. Any combination of fields can be included.

**Request body:**

| Field          | Type     | Required | Description                                                                     |
| -------------- | -------- | -------- | ------------------------------------------------------------------------------- |
| `name`         | string   | No       | New display name                                                                |
| `model`        | string   | No       | New model identifier                                                            |
| `allowedTools` | string[] | No       | List of tool IDs the agent can use (**admin only**)                             |
| `pluginConfig` | object   | No       | Plugin configuration, e.g. `{ "allowed_paths": ["/data/hr"] }` (**admin only**) |

**Example ‚Äî update permissions:**

```json
{
  "allowedTools": ["pinchy_ls", "pinchy_read"],
  "pluginConfig": {
    "allowed_paths": ["/data/hr-policies"]
  }
}
```

**Response:** The updated agent object.

**Errors:**

- `400` ‚Äî cannot change permissions for personal agents
- `401` ‚Äî not authenticated
- `403` ‚Äî only admins can change `allowedTools` or `pluginConfig`
- `404` ‚Äî agent not found

### `DELETE /api/agents/:id`

Delete an agent. **Admin only.** Personal agents (auto-created Smithers) cannot be deleted.

**Response:** `200 OK`

```json
{
  "success": true
}
```

**Errors:**

- `400` ‚Äî attempting to delete a personal agent
- `401` ‚Äî not authenticated
- `403` ‚Äî not an admin
- `404` ‚Äî agent not found

## Settings

### `GET /api/settings`

Get all application settings. Encrypted values (such as API keys) are masked in the response.

**Response:**

```json
[
  { "key": "default_provider", "value": "anthropic", "encrypted": false },
  { "key": "anthropic_api_key", "value": "--------", "encrypted": true }
]
```

### `POST /api/settings`

Update a setting.

**Request body:**

| Field   | Type   | Required | Description   |
| ------- | ------ | -------- | ------------- |
| `key`   | string | Yes      | Setting key   |
| `value` | string | Yes      | Setting value |

Settings with `api_key` in the key name are automatically encrypted at rest.

**Response:** `{ "success": true }`

## LLM Providers (admin only)

### `GET /api/settings/providers`

List configured LLM providers. Credentials are masked.

**Response:**

```json
[
  { "provider": "anthropic", "configured": true, "isDefault": true },
  { "provider": "openai", "configured": true, "isDefault": false }
]
```

### `POST /api/settings/providers`

Add or update a provider.

**Request body:**

| Field       | Type    | Required | Description                                 |
| ----------- | ------- | -------- | ------------------------------------------- |
| `provider`  | string  | Yes      | Provider identifier                         |
| `apiKey`    | string  | Cond.    | API key (required for cloud providers)      |
| `baseUrl`   | string  | Cond.    | URL (required for `ollama-local`)           |
| `isDefault` | boolean | No       | Mark as the default provider for new agents |

Pinchy validates the credentials by calling the provider's `/models` endpoint before persisting. API keys are encrypted at rest with AES-256-GCM.

### `DELETE /api/settings/providers?provider=<id>`

Remove a provider. Any agent still using a model from this provider will fail to start chats until reassigned.

### `GET /api/providers/models`

List available models across all configured providers. Populates the model dropdown in agent settings.

**Response:**

```json
{
  "models": [
    {
      "id": "claude-sonnet-4-6",
      "provider": "anthropic",
      "displayName": "Claude Sonnet 4.6",
      "toolUse": true
    }
  ]
}
```

The response is cached for one hour for cloud providers. Local Ollama is always fetched live.

## Telegram Channels

Connect agents to Telegram bots so users can chat with them from their phone. See [Set Up Telegram](/guides/telegram-setup/) for the end-to-end flow.

### `GET /api/agents/:agentId/channels/telegram`

Get the Telegram bot configuration for an agent. **Admin only.**

**Response (not configured):**

```json
{ "configured": false, "mainBotConfigured": true }
```

**Response (configured):**

```json
{ "configured": true, "hint": "8a2f", "mainBotConfigured": true }
```

`hint` is the last 4 characters of the bot token for visual verification. `mainBotConfigured` indicates whether the global Smithers/main bot is set up ‚Äî a prerequisite before any other agent can connect to Telegram.

### `POST /api/agents/:agentId/channels/telegram`

Connect an agent to a Telegram bot. **Admin only.** The main bot must be configured first (exception: the main bot itself).

**Request body:**

| Field      | Type   | Required | Description                 |
| ---------- | ------ | -------- | --------------------------- |
| `botToken` | string | Yes      | Bot token from `@BotFather` |

**Response:**

```json
{ "botUsername": "support_pinchy_bot", "botId": 123456789 }
```

**Errors:**

- `400` ‚Äî invalid or non-working bot token
- `409` ‚Äî `telegram_not_configured` (main bot missing), or the bot token is already used by another agent

### `DELETE /api/agents/:agentId/channels/telegram`

Disconnect a Telegram bot from an agent. **Admin only.** Personal (Smithers) agents cannot be disconnected individually ‚Äî use the "Remove Telegram for everyone" action instead.

### `GET /api/settings/telegram`

Get the current user's Telegram link status. Available to any authenticated user.

**Response:**

```json
{ "linked": true, "channelUserId": "123456789" }
```

### `POST /api/settings/telegram`

Link the current user's Telegram account using a pairing code received from the bot.

**Request body:**

| Field  | Type   | Required | Description                                     |
| ------ | ------ | -------- | ----------------------------------------------- |
| `code` | string | Yes      | Pairing code shown in Telegram after DM-ing bot |

**Response:** `{ "linked": true, "telegramUserId": "..." }`

**Errors:**

- `400` ‚Äî invalid or expired pairing code

### `DELETE /api/settings/telegram`

Unlink the current user's Telegram account.

### `GET /api/settings/telegram/bots`

List all Telegram bots connected across agents, with the agent each bot belongs to. **Admin only.**

### `DELETE /api/settings/telegram/all`

Remove Telegram from every agent at once. **Admin only.** This is the backend for the "Remove Telegram for everyone" flow in Settings ‚Üí Telegram.

## Integrations (admin only)

External system integrations ‚Äî currently **Odoo**. See [Integrations](/concepts/integrations/) for the conceptual overview and [Connect Odoo](/guides/connect-odoo/) for the setup guide.

### `GET /api/integrations`

List all integration connections with masked credentials.

### `POST /api/integrations`

Create a new integration connection.

**Request body (Odoo):**

```json
{
  "type": "odoo",
  "name": "Production Odoo",
  "description": "Main company instance",
  "credentials": {
    "url": "https://odoo.example.com",
    "db": "production",
    "login": "api-user@example.com",
    "apiKey": "..."
  }
}
```

Credentials are encrypted at rest with AES-256-GCM.

**Response:** `201 Created` ‚Äî the created connection with masked credentials.

**Errors:**

- `400` ‚Äî validation failed (bad URL, missing fields) or the URL resolves to a disallowed address (SSRF guard)

### `GET /api/integrations/:connectionId`

Get a single connection (credentials masked).

### `PATCH /api/integrations/:connectionId`

Update a connection's name, description, or credentials.

### `DELETE /api/integrations/:connectionId`

Delete a connection. Agents that reference it lose access immediately.

### `POST /api/integrations/:connectionId/sync`

Re-probe the external system: rediscover available models and recheck access rights. Permissions that no longer apply are removed.

### `POST /api/integrations/:connectionId/test`

Test credentials against the external system without persisting changes. Used by the setup wizard for live feedback.

### `GET /api/agents/:agentId/integrations`

Get an agent's integration permissions: which connections are enabled, access level (read-only, read-write, full, custom), and which data models are allowed.

### `PATCH /api/agents/:agentId/integrations`

Update an agent's integration permissions.

## Audit Trail (admin only)

### `GET /api/audit`

Retrieve a paginated, filterable audit log. **Admin only.**

**Query parameters:**

| Parameter   | Type   | Default | Description                                             |
| ----------- | ------ | ------- | ------------------------------------------------------- |
| `page`      | number | 1       | Page number                                             |
| `limit`     | number | 50      | Entries per page                                        |
| `eventType` | string | ‚Äî       | Filter by event type (e.g., `auth.login`, `tool.shell`) |
| `actorId`   | string | ‚Äî       | Filter by user ID                                       |
| `from`      | string | ‚Äî       | Start date (ISO 8601)                                   |
| `to`        | string | ‚Äî       | End date (ISO 8601)                                     |

**Response:**

```json
{
  "entries": [
    {
      "id": 1,
      "timestamp": "2026-02-21T10:00:00.000Z",
      "actorType": "user",
      "actorId": "user-uuid",
      "actorName": "Alice",
      "actorDeleted": false,
      "eventType": "auth.login",
      "resource": "user:user-uuid",
      "resourceName": "Alice",
      "resourceDeleted": false,
      "detail": {},
      "rowHmac": "sha256-hex-string"
    }
  ],
  "total": 142,
  "page": 1,
  "limit": 50
}
```

**Errors:**

- `401` ‚Äî not authenticated
- `403` ‚Äî not an admin

### `GET /api/audit/verify`

Verify the integrity of audit log entries by recomputing HMAC signatures. **Admin only.**

**Query parameters:**

| Parameter | Type   | Default | Description                                      |
| --------- | ------ | ------- | ------------------------------------------------ |
| `fromId`  | string | ‚Äî       | Start verification from this entry ID (optional) |
| `toId`    | string | ‚Äî       | End verification at this entry ID (optional)     |

**Response:**

```json
{
  "valid": true,
  "totalChecked": 142,
  "invalidIds": []
}
```

If tampered entries are found, `valid` is `false` and `invalidIds` contains the IDs of entries with mismatched signatures.

**Errors:**

- `401` ‚Äî not authenticated
- `403` ‚Äî not an admin

### `GET /api/audit/event-types`

List the distinct event types present in the current audit log. Useful for populating filter dropdowns without enumerating the full schema. **Admin only.**

**Response:**

```json
{
  "eventTypes": [
    "auth.login",
    "auth.failed",
    "agent.created",
```

Change your own password.

**Request body:**

| Field             | Type   | Required | Description                     |
| ----------------- | ------ | -------- | ------------------------------- |
| `currentPassword` | string | Yes      | Current password                |
| `newPassword`     | string | Yes      | New password (min 8 characters) |

**Errors:**

- `400` ‚Äî missing fields or new password too short
- `401` ‚Äî not authenticated or current password incorrect
